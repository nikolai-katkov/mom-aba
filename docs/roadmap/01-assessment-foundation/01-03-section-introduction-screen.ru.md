# 01-03: Экран введения в раздел

Статус: Черновик

## Описание

Обучающий экран, показываемый перед первой оценкой в разделе. Объясняет, что представляет собой категория навыков, почему она важна и что родитель будет делать дальше. Использует прогрессивное раскрытие: краткое объяснение видно по умолчанию, полное объяснение скрыто за кнопкой "Подробнее". Место для видео показывает статичное изображение-заглушку в MVP.

## Критерии приёмки для пользователя

- [ ] Краткое объяснение видно по умолчанию: 3--5 пунктов, охватывающих определение, практическую ценность и следующие шаги
- [ ] "Подробнее" разворачивает полное объяснение (теория, примеры, распространённые ошибки)
- [ ] Сворачивание "Подробнее" скрывает полное объяснение
- [ ] Место для видео отображает статичное изображение-заглушку с надписью "Видео скоро будет"
- [ ] Кнопка "Начать" переходит к списку критериев
- [ ] Кнопка "Пропустить" также переходит к списку критериев (тот же пункт назначения, другое отслеживание намерения)
- [ ] Контент специфичен для раздела (контент МАНД для МАНД)

## Системные критерии приёмки

- [ ] Маршрут: `/sections/:sectionId/intro`
- [ ] Контент введения загружается из данных раздела (не захардкожен в компоненте)
- [ ] Различие "Пропустить" и "Начать" отслеживается в состоянии (для готовности к аналитике в будущем, аналитика отсутствует в MVP)
- [ ] Если пользователь уже прошёл введение, возврат назад показывает его, но не блокирует продвижение

## Критерии приёмки для разработки

- [ ] Прогрессивное раскрытие использует CSS-переходы (не JS-анимацию)
- [ ] CSS Modules только с дизайн-токенами
- [ ] Адаптивность для всех профилей устройств
- [ ] Юнит-тест: поведение переключателя "Подробнее"
- [ ] E2E-тест: навигация из списка разделов, проверка отображения контента, нажатие "Начать" ведёт к списку критериев
