# 01-01: Модель данных оценки

Статус: Черновик

## Описание

Определить TypeScript-интерфейсы и начальные данные для иерархии оценки. Две отдельные модели: данные оценки (разделы, критерии, условия) и данные тренировки (руководства, шаги). Заполнить раздел МАНД 5 критериями и соответствующим тренировочным контентом. ТАКТ включён только как метаданные (заглушка).

## Критерии приёмки для пользователя

- [ ] Раздел МАНД содержит 5 критериев с понятными названиями, описаниями, условиями и примерами
- [ ] Каждый критерий имеет тип оценивания (TCT, NAB, KOM, NOV), который UI может использовать для выбора режима ввода
- [ ] Раздел ТАКТ существует как заглушка с названием и статусом "скоро будет"
- [ ] Тренировочный контент для каждого критерия МАНД включает пошаговое руководство

## Системные критерии приёмки

- [ ] TypeScript-интерфейсы: `Section`, `Criterion`, `ScoringType`, `CriterionStatus`, `TrainingContent`
- [ ] Перечисление `ScoringType`: `TCT`, `NAB`, `KOM`, `NOV`
- [ ] Перечисление `CriterionStatus`: `NotStarted`, `InProgress`, `Completed`
- [ ] Данные оценки и тренировки -- отдельные структуры данных, связанные по ID критерия
- [ ] Балл хранится как числовое значение (0, 0.5, 1) внутренне, даже если UI показывает бинарный выбор Да/Нет (см. [Вопросы PM: Модель оценивания](../../knowledge/pm-questions-scoring-model.ru.md))
- [ ] Структура поддерживает добавление уровней II/III и дополнительных разделов без нарушения совместимости
- [ ] Модель измерений развития: Самостоятельность, Генерализация, Объём репертуара

## Критерии приёмки для разработки

- [ ] Все интерфейсы экспортируются из barrel-файла (`src/types/` или `src/data/`)
- [ ] Начальные данные в отдельных файлах, не встроенные в компоненты
- [ ] Юнит-тесты проверяют полноту начальных данных (все 5 критериев МАНД присутствуют, все поля заполнены)
- [ ] Юнит-тесты проверяют корректность типов начальных данных
- [ ] Отсутствие типов `any`
