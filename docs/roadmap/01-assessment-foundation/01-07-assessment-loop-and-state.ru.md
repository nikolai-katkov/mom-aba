# 01-07: Цикл оценки и управление состоянием

Статус: Черновик

## Описание

Связать полный цикл оценки: Оценка > (если Нет) Тренировка > Переоценка > повтор. Обеспечить согласованность переходов состояний, сохранение прогресса между сеансами и наличие у пользователя ясного пути вперёд. Эта задача охватывает сквозное управление состоянием и навигационный поток, связывающий экраны с 01-02 по 01-06.

## Критерии приёмки для пользователя

- [ ] После завершения тренировки и нажатия "Повторить" пользователь видит экран оценки того же критерия (не другого)
- [ ] После нажатия "Да" при переоценке критерий обновляется до статуса "Завершено" и пользователь возвращается к списку критериев
- [ ] Прогресс на экране списка разделов отражает актуальное состояние (без устаревших данных)
- [ ] Закрытие и повторное открытие приложения сохраняет весь прогресс (состояния критериев, прогресс разделов)
- [ ] Навигация назад в любой точке не теряет прогресс
- [ ] Переходы статуса раздела: Не начато -> В процессе (первая оценка) -> Завершено (все 5 выполнены)

## Системные критерии приёмки

- [ ] Состояние хранится в localStorage (слой персистенции MVP)
- [ ] Форма состояния: словарь ID критериев к `{ status: CriterionStatus, score: number }`
- [ ] Обновления состояния атомарны (без частичных записей, которые могут повредить данные)
- [ ] Навигация браузера вперёд/назад работает корректно в цикле оценки
- [ ] Защита маршрутов: доступ к оценке/тренировке для несуществующего критерия перенаправляет к списку критериев
- [ ] Нет тупиковых состояний: каждый экран имеет хотя бы один вариант навигации вперёд

## Критерии приёмки для разработки

- [ ] Управление состоянием через React context или пользовательский хук (без пробрасывания пропсов через 4+ уровня)
- [ ] Единый источник истины для состояния оценки (один хук/хранилище, не разрозненные вызовы useState)
- [ ] Юнит-тесты для переходов состояний: NotStarted -> InProgress -> Completed
- [ ] Юнит-тесты для персистенции localStorage (запись при изменении, чтение при монтировании)
- [ ] Юнит-тест: повреждённый/отсутствующий localStorage откатывается к чистому начальному состоянию
- [ ] E2E-тест: полный цикл -- оценка "Нет" -> тренировка -> повтор -> оценка "Да" -> список критериев обновлён
- [ ] E2E-тест: закрытие и повторное открытие приложения, проверка сохранения состояния
